## 프로젝트 개요

**목적:**
사용자가 특정 맛집에 대해 여러 포스팅을 볼 때, 서로 다른 의견과 평가를 한눈에 파악할 수 있도록 수집된 정보를 분석하고 요약하여 제공합니다.

**핵심 가치:**
- 사용자가 한정된 시간에 맛집의 다양한 정보를 효율적으로 파악 가능
- 다양한 포스팅의 서로 다른 평가를 비교 분석하여 신뢰성 있는 정보 제공
- 관리자가 맛집 정보를 등록하고 관리할 수 있는 시스템 제공

## 핵심 기능

### 1. 데이터 수집 (Data Collection)

#### 1.1 네이버 포스팅 수집
- **검색 방식:**
  - 맛집 이름으로 네이버 검색 수행
  - 검색 결과에서 포스팅(블로그, 인스타그램 등) 수집
  - 동일한 맛집 여러 포스팅 식별 (이름, 위치, 전화번호 등)

- **정보 수집 항목:**
  - **맛집 기본 정보:**
    - 이름, 주소, 전화번호, 메뉴 종류
    - 영업 시간, 휴무일
    - 가격대
  - **포스팅 특정 평가:**
    - 추천 메뉴/취향
    - 가성비 평가
    - 분위기/인테리어
    - 서비스 만족도
    - 개인적 취향/감상
  - **공통 평가:**
    - 맛 (별점)
    - 서비스 (별점)
    - 가성비 (별점)
    - 청결도 (별점)
    - 총점 (별점)

#### 1.2 데이터 분류
- **공통 평가 항목 (Shared Attributes):**
  - 맛, 서비스, 가성비, 청결도, 총점 (리뷰 포스팅에 포함된 모든 별점)

- **개인별 평가 항목 (Personal Attributes):**
  - 추천 메뉴/취향
  - 가성비 판단 기준
  - 분위기/인테리어 선호
  - 서비스 만족도 개인적 의견
  - 개인적 감상 및 경험

### 2. 정리 및 분석 (Analysis & Summary)

#### 2.1 공통점 (DO) 분석
수집된 포스팅에서 모두 동의하는 항목을 도출:

- **공통 평가:**
  - 평균 별점 계산
  - 최소/최대/중앙값
  - 다수결 기준 평가

- **공통 인사이트:**
  - 모든 포스팅이 동의하는 메뉴/특징
  - 모두 긍정적으로 언급한 장점
  - 모두 부정적으로 언급한 단점

#### 2.2 차이점 (DONT) 분석
포스팅 간의 다른 의견을 분석:

- **개인별 취향:**
  - 추천 메뉴/취향별 차이
  - 가성비 판단 기준 차이
  - 분위기 선호도 차이

- **의견 차이:**
  - 긍정/부정 평가의 차이
  - 이유 제시의 차이
  - 경험에 따른 판단 차이

#### 2.3 요약 리포트 작성
사용자가 이해하기 쉬운 형태로 정리:

- **맛집 개요:**
  - 이름, 주소, 연락처, 영업 시간
  - 메뉴 정보 및 가격대

- **종합 평점:**
  - 공통 평가 항목별 평균 별점
  - 총점 (가중 평균 또는 단순 평균)

- **DO (공통점):**
  - 모두 동의하는 장점
  - 추천하는 이유

- **DONT (차이점):**
  - 서로 다른 의견 요약
  - 개인별 취향 차이

- **주의사항:**
  - 개인적인 의견 차이에 주의
  - 환경/상황에 따라 평가가 달라질 수 있음

### 3. 플랫폼 구축 (Platform)

#### 3.1 관리자 기능
- **맛집 등록:**
  - 맛집 이름, 주소, 전화번호, 메뉴 등 기본 정보 입력
  - 복수의 포스팅 수집 URL 등록

- **포스팅 관리:**
  - 수집된 포스팅 목록 확인
  - 포스팅 추가/수정/삭제
  - 분석 결과 확인

- **데이터 관리:**
  - 수집된 맛집 목록 관리
  - 검색 기능
  - 필터링 기능

#### 3.2 사용자 인터페이스
- **맛집 검색:**
  - 맛집 이름으로 검색
  - 검색 결과 목록 제공

- **맛집 상세 페이지:**
  - 맛집 개요 정보
  - 종합 평점
  - DO (공통점) 섹션
  - DONT (차이점) 섹션
  - 주의사항
  - 참조 포스팅 목록

## 데이터 수집 전략

### 1. 웹 스크래핑 접근 방식

#### 선택 전략
**네이버 블로그 스크래핑 (선호)**
- 장점: 콘텐츠가 풍부하고 평가 정보가 자세함
- 단점: 네이버 차단 가능성, 구조 변경 필요
- 구현 방식: Cheerio + Axios

**인스타그램 URL 수집 (선택)**
- 장점: 시각적 정보 수집 가능
- 단점: API 제한, 수집 복잡도 높음
- 구현 방식: Instagram Graph API 또는 웹 스크래핑

#### 수집 흐름
```
1. 사용자가 맛집 이름 입력
   ↓
2. 네이버 검색 수행 (검색 엔진 또는 직접 접속)
   ↓
3. 검색 결과에서 맛집 URL 수집
   ↓
4. 각 URL에서 포스팅 수집
   - 블로그 포스팅
   - 카페 포스팅
   - 인스타그램 (가능한 경우)
   ↓
5. 포스팅에서 평가 정보 추출
   - 별점 (taste, service, value, cleanliness)
   - 텍스트에서 평가 정보 파싱
   ↓
6. 데이터베이스 저장
   ↓
7. 분석 서비스 실행 (DO/DONT 분석)
   ↓
8. 요약 리포트 생성
```

### 2. 스크래핑 예외 처리

- **네이버 차단:** User-Agent 헤더 설정, 요청 간격 조절
- **동적 콘텐츠:** 가끔 필요할 수 있음 (Puppeteer 고려)
- **로그인 필요:** 대부분의 포스팅은 공개로 가정
- **CAPTCHA:** 로봇 차단 시 대체 수집 방안 검토

### 3. 데이터 검증

- 중복 포스팅 필터링 (URL 기반)
- 날짜 기반 최신 포스팅 우선
- 평점 범위 검증 (1-5)
- 필수 필드 존재 확인
