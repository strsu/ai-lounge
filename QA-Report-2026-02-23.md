## 문제점 및 개선 사항

### 주요 문제

#### 1. 데이터베이스 마이그레이션 실패
**상태:** ⚠️ 보고됨  
**심각도:** 중간

- **문제:** nabijiyo-migrate Job이 모두 Error 상태
- **에러 메시지:**
  ```
  P1001: Can't reach database server at `postgres:5432`
  ```
- **영향:**
  - 배포 시 자동 마이그레이션이 실패
  - 수동으로 `prisma db push` 실행 필요
  - 배포 자동화가 완전하지 않음

- **원인 분석:**
  - Job 시작 시점에 postgres가 완전히 준비되지 않았을 가능성
  - Job의 데이터베이스 연결 타임아웃 설정이 짧을 가능성

#### 2. Sub-agent 통신 불능
**상태:** ⚠️ 지속 중  
**심각도:** 높음

- **문제:** pm/developer 에이전트와 sessions_send 불가
- **에러:** ws://127.0.0.1:18789 Gateway timeout
- **영향:**
  - 작업 위임 불가
  - 협업 프로세스 차단
  - QA 검증에 제약

### 권장 사항

#### 1. 마이그레이션 개선
1. **Job 재시작 메커니즘 추가:**
   - 실패 시 자동 재시작 로직 (restartPolicy: OnFailure)
   - 최대 재시작 횟수 제한 (backoffLimit)

2. **데이터베이스 연결 타임아웃 증가:**
   - connection timeout을 30초로 증가
   - PostgreSQL connection pool 설정 검토

3. **초기화 순서 개선:**
   - postgres가 완전히 준비된 후에 migrate Job 시작
   - initContainer 또는 wait-for-it-s 도구 사용

#### 2. UX 개선
1. **로딩 상태 표시:**
   - 스크래핑 중 로딩 인디케이터
   - API 요청 중 스피너

2. **에러 메시지 개선:**
   - 사용자 친화적인 에러 메시지
   - 에러 유형별 안내

3. **빈 상태 처리:**
   - 검색 결과 없음 메시지
   - 포스팅 없음 안내

#### 3. 예외 처리 강화
1. **네트워크 에러 처리:**
   - 재시도 로직 (exponential backoff)
   - 타임아웃 설정

2. **스크래핑 실패 처리:**
   - 네이버 차단 감지
   - 대체 수집 방안 (User-Agent 로테이션)

3. **데이터 검증:**
   - 스크래핑 데이터 유효성 검사
   - 중복 데이터 필터링 강화
