# 맛집 포스팅 요약 플랫폼 기획 - 2026-02-08

## 작업 개요
네이버 맛집 포스팅 요약 플랫폼 기획 수행

## 진행 상황

### 1. 요구사항 분석 및 기능 명세서 작성
- 핵심 기능 정의:
  - 데이터 수집: 네이버에서 동일한 맛집의 여러 포스팅 수집, 공통 평가 vs 개인별 평가 분류
  - 정리 및 분석: DO(공통점)와 DONT(차이점) 구체화, 사용자 친화적 요약 리포트 생성
  - 플랫폼 구축: 자체 웹사이트 운영(API/SNS 게시 아님), 관리자용 웹 대시보드

- 공통 평가 항목 vs 개인별 평가 항목 분류:
  - 공통 평가: 맛, 서비스, 가성비, 청결도, 총점
  - 개인별 평가: 추천 메뉴/취향, 가성비 판단 기준, 분위기/인테리어 선호, 서비스 만족도 개인적 의견, 개인적 감상 및 경험

### 2. 기술 스택 선정
**프론트엔드**:
- Next.js 14.x (App Router)
- TypeScript 5.x
- Tailwind CSS
- Chart.js (평점 그래프)

**백엔드**:
- Next.js API Routes
- PostgreSQL + Prisma ORM

**데이터 수집**:
- Cheerio (HTML 파싱)
- Axios (HTTP 클라이언트)
- 네이버 블로그, 지식인, 인스타그램

**배포**:
- Docker
- Kubernetes
- ArgoCD (GitOps)

### 3. 시스템 아키텍처 설계
- 사용자 인터페이스 (검색/상세/관리자 대시보드)

### 4. 데이터 수집 전략

- 중복 포스팅 필터링, 평점 추출, 데이터 검증
